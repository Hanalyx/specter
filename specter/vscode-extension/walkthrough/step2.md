# Close the gaps

Open any generated spec — it will look like this:

```yaml
spec:
  id: user-create
  gap: true              # ← human review needed
  constraints:
    - id: C-01
      description: "POST /users accepts email and password"
  acceptance_criteria:
    - id: AC-01
      description: ""    # ← empty, needs your intent
```

## Pass this prompt to your AI assistant

```
Here is a draft spec generated by Specter's reverse compiler:

[paste the spec]

Please:
1. Fill in all empty acceptance_criteria descriptions with clear, testable behavior
   (e.g. "Valid email + password creates user and returns 201 with JWT token")
2. Add any constraints that are obviously missing based on the context
3. Add `references_constraints` arrays to each AC linking back to the C-XX IDs it validates
4. Remove the `gap: true` flag once all gaps are filled
5. Keep all existing IDs (C-01, AC-01, etc.) unchanged
6. Use MUST/MUST NOT language for constraints per RFC 2119

Return only the completed YAML.
```

## Validate the result

After AI edits, run:

```bash
specter parse specs/user-create.spec.yaml
```

```
PASS specs/user-create.spec.yaml — user-create@1.0.0
```

The extension also validates in real-time — schema errors appear in the Problems panel as you type.

> **Tip:** Use **Specter: Copy Spec Context for AI** (right-click any `.spec.yaml`) to copy the spec as a structured AI prompt preamble before asking your AI to implement it.
